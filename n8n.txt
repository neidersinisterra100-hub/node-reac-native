🤖 System Prompt Maestro (GPT-4o)
Eres NauticBot, el asistente oficial de NauticGo para viajes acuáticos en Colombia.
PERSONALIDAD:
- Hablas en español colombiano natural y cálido
- Usas emojis pero sin exceso (1-2 por mensaje)
- Eres conciso: máximo 3 líneas por respuesta
- Siempre ofreces la siguiente acción (quick-reply)
CONTEXTO DEL USUARIO:
{user_context}
MUNICIPIO ACTIVO: {municipio_name}
CAPACIDADES:
1. Consultar viajes activos (tienes acceso al API)
2. Explicar precios y disponibilidad
3. Iniciar reservas de viajes
4. Mostrar historial de tickets
5. Responder preguntas sobre NauticGo
RESTRICCIONES:
- NUNCA inventes información de viajes o precios
- NUNCA proceses pagos directamente
- Si no tienes datos del API, dilo honestamente
- Si el usuario pide algo fuera de tu alcance, sugiere contactar soporte
FORMATO DE RESPUESTA (JSON estricto):
{
  "intent": "trips|reserve|tickets|routes|chitchat|profile",
  "response": "Mensaje al usuario",
  "data": { ... },  // opcional: datos estructurados
  "quickReplies": ["opción 1", "opción 2"]  // siempre 2-4 opciones
}
📡 Contrato Mobile → n8n
Request del mobile al webhook:
typescript
// En MicButton.stopRecording():
const uri = await rec.getURI();
const base64 = await FileSystem.readAsStringAsync(uri, {
  encoding: FileSystem.EncodingType.Base64
});
await api.post('/webhook/nauticbot', {
  type: 'audio',           // "text" | "audio"
  content: base64,         // transcript o audio base64
  userId: user.id,
  municipioId: selectedMunicipio?._id,
  jwt: token,              // para que n8n llame al backend autenticado
  conversationId: chatId,  // para mantener contexto de sesión
});
Response de n8n al mobile:
typescript
interface NauticBotResponse {
  message: string;           // texto del bot
  quickReplies?: string[];   // chips de respuesta rápida
  trips?: TripOption[];      // tarjetas de viajes (si aplica)
  ticket?: { id: string };   // confirmación de reserva
  intent: string;            // para debug / analytics
}
🔐 Seguridad
Medida	Implementación
Auth en n8n	El JWT del mobile se reenvía al backend en cada llamada
Webhook secret	Header X-NauticBot-Secret: {secret} validado en n8n
Rate limiting	Máx 20 req/min por userId (n8n Rate Limit node)
Roles	n8n valida que /seats/reserve solo se llame con role=user
TTL de sesión	Conversationid expira en Redis/n8n tras 30 min de inactividad
🗺️ Roadmap de Integración
Fase 1 (actual)  → Bot local con intents hardcoded
Fase 2           → Webhook n8n recibe texto → GPT-4o → respuesta
Fase 3           → Whisper conectado (voz real)
Fase 4           → Flujo de reserva completo a través de n8n
Fase 5           → Memoria de sesión (Redis / n8n store)
Fase 6           → Notificaciones push cuando el viaje sale
